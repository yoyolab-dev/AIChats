# ADR-005: 消息 Markdown 渲染与安全过滤

## 状态
✅ 已采纳

## 上下文

用户消息支持富文本格式（Markdown），但需要防止 XSS 攻击，同时确保渲染后的 HTML 安全。

## 考虑的方案

- **前端渲染**: 将 Markdown 发送到前端由浏览器渲染，可以使用 `marked` 等库。但不同客户端行为不一致，且无法在管理后台统一查看。
- **后端渲染 + 过滤**: 服务器端将 Markdown 转为 HTML，并通过 `sanitize-html` 过滤危险标签/属性。

## 决策

在 **后端** 完成 Markdown 到 HTML 的转换与安全过滤。

### 实现

- 使用自定义 `renderMarkdown` 工具（集成 `marked` + `sanitize-html`）。
- 在消息创建/更新时，同时保存原始内容 `content` 和渲染后的 `contentHtml`。
- 前端直接显示 `contentHtml`（节省客户端计算，一致性高）。

### 理由

- 统一渲染逻辑，避免多端重复实现。
- 可集中控制允许的 HTML 标签和属性（如只保留 `a`、`code`、`pre` 等）。
- 方便管理后台直接查看安全渲染后的消息。

## 后果

- 存储略微增加（同时保存原始和渲染内容）。
- 修改渲染规则需要重新渲染历史消息（可通过后台任务处理）。
- 若需支持实时协作编辑，需要重新考虑渲染策略。
