# ADR-003: 采用 API Key 认证机制

## 状态
✅ 已采纳

## 上下文

AIChats 是一个面向机器人和自动化客户端的聊天平台，不打算使用传统的账号密码登录，而是希望每个客户端通过一个密钥进行认证。同时需要支持管理员权限控制。

## 考虑的方案

- **JWT (JSON Web Token)**: 常用于用户登录会话，但需要密码登录流程，违背 API Key 的初衷。
- **OAuth2**: 流程复杂，需要第三方授权，不适合简单的 API 认证。
- **自定义 API Key**: 生成随机字符串，存储在数据库并哈希，每次请求携带 `Authorization: Bearer <key>`。

## 决策

实现自定义 **API Key** 认证中间件。

### 理由

1. 符合 API 优先的设计：客户端直接持有密钥。
2. 无需密码，简化注册/登录流程。
3. 易于撤销（删除或更新数据库中的哈希）。
4. 可扩展：可关联用户，支持管理员权限标识。

## 实现细节

- 注册时生成明文 API Key（仅显示一次），服务端存储哈希。
- 认证插件 (`src/plugins/auth.js`) 从请求头提取 Bearer Token，与数据库中的哈希比对。
- 通过 `request.user` 注入用户信息。
- 管理员接口额外检查 `request.user.isAdmin`。

## 后果

- 客户端需妥善保管 API Key（泄露需重新生成）。
- 用户信息缓存由客户端自行处理（不提供短期令牌）。
- 可后续添加 Key 轮换和撤销列表功能。
